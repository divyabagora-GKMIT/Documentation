Test Report - Employee Management System

TP-01: Verify User Authentication Functionality

Test Plan ID: TP-01 Title: Verify User Authentication Functionality

Acceptance Criteria

  • User must be able to log in using valid email/username and password.
  • Password must be encrypted and securely validated.
  • Failed login attempts must display an error message.
  • Unauthorized users must not access restricted pages.
  • Account should lock after a predefined number of failed attempts.

TP-02: Validate Employee Data CRUD Operations

Test Plan ID: TP-02 Title: Validate Employee Data CRUD Operations

Acceptance Criteria

  • Admin/HR can add, edit, update, and delete employee records (CRUD operations).
  • Employee ID must be unique.
  • Only Admin/HR roles can modify employee details.
  • Successful operations must display confirmation messages.
  • Employee profile view must display accurate information.

TP-03: Validate Department Creation & Employee Association

Test Plan ID: TP-03 Title: Validate Department Creation & Employee Association

Acceptance Criteria

  • Department code must be unique.
  • Admin/HR can add and update department details.
  • Employees can be associated correctly with their departments.
  • System must display the department list with associated employees.

TP-04: Validate Project & Employee Assignment Functionality

Test Plan ID: TP-04 Title: Validate Project & Employee Assignment Functionality

Acceptance Criteria

  • Every project must have at least one assigned employee.
  • Employees must not be assigned to inactive projects.
  • Project data (name, ID, description, dates) must be stored correctly.
  • Project list must show employees and assigned roles.

TP-05: Validate Search Across Employees, Departments, and Projects

Test Plan ID: TP-05 Title: Validate Search Across Employees, Departments, and Projects

Acceptance Criteria

  • Search must work using name, email, or project title.
  • System must display matching results with basic info.
  • If no record is found, system should show “No records found.”

TP-06: Verify Role-Based Access Control (RBAC)

Test Plan ID: TP-06 Title: Verify Role-Based Access Control

Acceptance Criteria

  • Employee role: View only their own profile.
  • HR/Admin role: View and manage all records.
  • Unauthorized actions must be denied and logged.
  • System must allow temporary role-switching (e.g., HR acting as Employee during leave request).
